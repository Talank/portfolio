Cheat Sheet

Every keybinding table from the course, in one scrollable page. Not a substitute for the lessons — a fast recall aid once you've actually done the labs. Notation: C-x = Control+x, M-x = Meta+x (Option, once Part 1 sets that up), s-x = Super/Cmd+x. Grows as each Part gets written; organized in the same order as the dashboard.

Part 0 — Orientation & Starting Emacs

KeysWhat it does
C-gCancel/abort whatever is currently happening — the universal panic button
C-x C-cQuit Emacs
C-h tLaunch the built-in interactive tutorial
C-h k then a keyDescribe exactly what that key sequence runs
C-h f / C-h vDescribe a function / describe a variable
emacs --daemonStart a persistent background Emacs server
emacsclient -cOpen a new GUI frame connected to the running daemon

Part 1 — The Mac Keybinding Model

Keys / SettingWhat it does
Option ↦ MetaDefault GUI Emacs.app mapping — Option is your M- key
mac-option-modifierElisp variable controlling what Option sends (nil = normal accented-char Option)
mac-right-option-modifierSet Right Option separately — keep it typing accented characters while Left Option is Meta
Terminal.app → "Use Option as Meta key"Required checkbox for M- to work in terminal Emacs
iTerm2 → Left/Right Option "Esc+"Same idea, set independently per side, unlike Terminal.app
Esc then a keyUniversal Meta fallback — works everywhere, no config needed
mac-command-modifierWhat Command sends (rebind away from macOS OS-level shortcuts if desired)

Part 4 — Customization & Config

Keys / ConceptWhat it does
~/.emacs.d/init.elYour init file — evaluated top to bottom, once, at startup

Part 7 — Terminal, Shell, Remote & Capstone

Keys / PathWhat it does
M-x shellReal system shell ($SHELL) as a genuine subprocess
M-x eshellEmacs's own Lisp-native shell — no subprocess, mixes Lisp into commands
C-x gMagit: open magit-status
s / u in MagitStage / unstage a hunk or file
c c then C-c C-cMagit: start a commit message, then confirm it
/ssh:user@host:/pathTRAMP: open a remote file over SSH as if local
/ssh:user@host|sudo:root@host:/pathTRAMP: chain SSH then sudo on the same connection
C-x d /ssh:user@host:/path/Dired on a remote directory — every operation runs remotely
