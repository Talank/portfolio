Part 1 — Math Foundations

Vectors, dot product, cosine similarity

a·b = Σ aᵢbᵢ = |a||b|cos(θ)    cos_sim(a,b) = (a·b) / (|a||b|)cosine similarity = dot product on UNIT-LENGTH vectors. Range [-1, 1]. This one identity powers word2vec, attention scores, and RAG retrieval.

Matrices & eigenvectors

(AB)ᵢⱼ = Σₖ AᵢₖBₖⱼ    Av = λv    A = UΣVᵀ (SVD)an eigenvector's direction is unchanged by A, only scaled by λ. PCA finds directions of maximum variance via the covariance matrix's eigenvectors.

Calculus & the chain rule

dz/dx = (dz/dy)(dy/dx)    ∇f = [∂f/∂x₁, …, ∂f/∂xₙ]    θ ← θ − η∇L(θ)backprop is the chain rule applied systematically through a computational graph, in reverse.

Probability & statistics

P(A|B) = P(B|A)P(A) / P(B)  (Bayes)    θ̂_MLE = argmax_θ Π p(xᵢ|θ)minimizing MSE ≡ maximizing likelihood under a Gaussian noise assumption. Minimizing cross-entropy ≡ maximizing likelihood under a categorical assumption.

Information theory

H(p) = −Σ p(x)log p(x)    CE(p,q) = −Σ p(x)log q(x)    KL(p‖q) = CE(p,q) − H(p)cross-entropy loss = entropy of the true distribution + KL divergence from prediction to truth. Training pushes KL toward 0.

Part 2 — Classical ML

Linear & logistic regression

ŷ = Xθ  (normal eq: θ = (XᵀX)⁻¹Xᵀy)    σ(z) = 1/(1+e⁻ᶻ)    ∂CE/∂z = ŷ − ythe "predicted minus actual" gradient shows up everywhere: logistic regression, softmax classification, and the transformer's output layer.

Model evaluation

precision = TP/(TP+FP)  recall = TP/(TP+FN)  F1 = 2·P·R/(P+R)  bias² + variance + noise = error
SituationPrioritize
Missing a positive is costly (fraud, disease)High recall
A false alarm is costly (spam-blocking a real email)High precision
Severe class imbalancePR-AUC over ROC-AUC; never bare accuracy
Comparing error types with different $ costCost-weighted threshold, not F1 (Part 8)

k-NN, trees, clustering

k-means: minimize Σᵢ ‖xᵢ − μ_{c(i)}‖²    entropy-based split gain (decision trees)

Part 3 — Deep Learning

Neurons & training

a = f(Wx+b)    v ← βv + (1−β)∇L  (momentum)    Adam: m,v bias-corrected moments, θ ← θ − η·m̂/(√v̂+ε)
LayerNorm(x) = γ·(x−μ)/√(σ²+ε) + β  (per-token, not per-batch)
NeedReach for
Nonlinearity, hidden layersReLU/GELU (dying-ReLU risk → GELU/Leaky)
Prevent co-adaptation / overfittingDropout (train only), weight decay, early stopping
Stabilize deep stacksResidual connections + LayerNorm (pre-norm is the modern default)
Variable-length sequences in a batchLayerNorm, not BatchNorm (batch-size independent)

Part 4 — NLP

TF-IDF(t,d) = tf(t,d) · log(N / df(t))    BPE: iteratively merge the most frequent adjacent symbol pair
skip-gram objective: maximize Σ log σ(u_c·v_w) + Σ log σ(−u_neg·v_w)  (negative sampling)
Bahdanau attention: e = score(s,hᵢ) → α = softmax(e) → context = Σαᵢhᵢ

Attention(Q,K,V) = softmax(QKᵀ/√dk)·V÷√dk keeps softmax out of its saturated region — variance of a raw dot product grows with dk.

Part 6 — LLM Engineering

Pretraining & scaling

C ≈ 6ND  (compute ≈ 6 × params × tokens)    compute-optimal: D ≈ 20N  (Chinchilla)

Fine-tuning

LoRA: ΔW = BA,  A: r×d_in, B: d_out×r, r≪d    forward: h = xW + (α/r)·xAᵀBᵀ  (W frozen)
MethodTrainsBest for
Full fine-tuneEvery parameterLarge curated dataset + ample compute + genuine need for broad behavior change
LoRASmall low-rank adapter (A, B)Default choice — cheap, swappable, resistant to catastrophic forgetting
QLoRALoRA adapter, base quantized to 4-bitLoRA's benefits when the base model doesn't fit in memory otherwise

Alignment

Bradley-Terry: P(chosen≻rejected) = σ(r(chosen)−r(rejected))
DPO loss = −log σ( β·[log(π_θ(y_w|x)/π_ref(y_w|x)) − log(π_θ(y_l|x)/π_ref(y_l|x))] )

temperature: softmax(logits/T)  T<1 sharpens, T>1 flattens

Part 7 — RAG & Agents

RAG

index-time: docs → chunks → embed → store    query-time: question → embed → top-k search → prompt → answer
SymptomLikely causeFix
recall@k lowRetrieval failureTune chunk size/overlap, embedding model, or add re-ranking

Agents

ReAct loop: Thought → Action → Observation → … → Final Answer

MCP: M frameworks × N tools → M+N  (one server per tool, one client per framework)
Question to askWhat it tells you
Does recall@k need fixing, or does generation need fixing?Retrieval vs. generation failure (RAG)
Is the objective loosely specified with real-world actions available?Reward-hacking / Goodhart risk — scope tools tightly
Is this action consequential and hard to reverse?Requires an approval gate, not autonomous execution
Is this content from a retrieved doc or tool output?Treat as untrusted — a real prompt-injection surface

"Which tool when" — the master table

GoalReach for
Teach a model new FACTS, or facts that change oftenRAG (retrieval), not fine-tuning
Teach a model a specific STYLE, FORMAT, or behaviorFine-tuning (LoRA by default)
Adjust behavior toward human preferences (helpful/harmless/honest)RLHF or DPO, after SFT
A task needs a real-world action, current data, or precise computationAn agent with a scoped tool, not a bigger model
A task is simple, well-defined, high-volumeRoute to a small/cheap model, not the frontier tier by default
A task is genuinely open-ended, ambiguous, high-stakes reasoningThe most capable tier available, reserved for exactly this
Base model fits in memory, need to fine-tuneLoRA
Base model does NOT fit in memoryQLoRA
A workflow has loops/branches decided at runtimeLangGraph (or a hand-rolled loop if simple enough)
A workflow is a fixed, linear pipelineA simple chain — no graph needed
An action is consequential and hard to reverseHuman-approval gate, regardless of model confidence
