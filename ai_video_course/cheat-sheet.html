Part 5 — Pipeline Engineering

ThingWhat it is
ffprobeInspect a media file's duration, streams, codecs without decoding it
ffmpeg -i in -map ... outDeterministic glue: extract, normalize, mux, caption-burn, concat
Content-addressed cache keySHA-256 hash of canonicalized stage inputs — identical inputs anywhere hit the same cached output
Cascade invalidationA changed upstream hash automatically changes every downstream key that depends on it — no manual bookkeeping
Atomic artifact writeWrite to a temp path, rename on success — a crash mid-write never leaves a corrupt "finished" artifact
The plan schemavoice + stages + gestures + ambient + params + unmatched, as one constrained document the director emits
Reject vs. repairEncoding mistakes get repaired (bounded loop); policy violations (unauthorized voice) get rejected outright, never repaired
Plan boldly, pay marginallyThe director always emits a complete plan; content-keyed caching means unchanged stages cost nothing to re-plan
Bounded replanningAlternative plans on failure, no repeated plan hashes, hard cap (e.g. 2) before escalating to a human
VRAM residency: max, not sumSequential batch stages need only the PEAK single-stage footprint; co-resident live-mode stages need the SUM
K-quants / AWQ / GPTQBlock quantization / importance-aware quantization — quality cost varies sharply by model type (diffusion tolerates least)
Golden-set gatingEvery quant/model swap is validated against a fixed golden set before it becomes default — never eyeballed once and shipped
Cost-per-MB evictionCache eviction ranks by remake cost per MB freed, with pins for protected entries — not plain LRU

Part 6 — Shipping It

ThingWhat it is
Content-hash uploadVerify magic bytes (not extension/content-type), hash at the door — the same hash the cache keys on
Resumable/chunked uploadOffset-tagged chunks; reconnect resumes from the last acked offset instead of byte zero
202 AcceptedJob creation returns instantly with a job_id — the response is a receipt, not a result
Idempotency-Key headerClient-generated key on submission; a retried request with a matching plan hash returns the SAME job, no duplicate work
SSE (Server-Sent Events)One-directional progress stream over plain HTTP — chosen over WebSockets (no duplex need) and polling (no interval to tune)
Last-Event-IDBrowser resends this on reconnect; server replays only the buffered gap, then continues live — no missed or duplicate events
Error envelope retryable fieldExplicit signal for retry-with-backoff vs. show-and-stop, instead of inferring policy from status codes
One source of truth (frontend)One plan object in memory; every panel (timeline, preview, cost) is a pure view of a slice of it
Optimistic vs. pessimistic UIOptimistic for locally-verifiable edits (drag); pessimistic/wait for anything the UI can't predict (NL edit)
Scoped, revocable consent grantPersonal/commercial/project scope + expiry + revoked flag — checked fresh on every generation, never cached
Provenance manifestSigned, tamper-evident record of what generated an asset — embedded AT generation time, never bolted on after
In-media watermarkRedundant to metadata — survives re-encoding/compression that strips sidecar metadata outright
Misuse review queuePublic-figure name matches + volume anomalies route to a human — narrow hard-block list, everything else reviewed, not auto-rejected
Audit trail minimizationKeep grant id + content hash + decision; skip raw scripts/media and unneeded fingerprinting by default

Part 7 — Evaluation & Capstone

ThingWhat it is
Proxy metricCheap, automated, runs on every render (sync offset, no-reference audio/frame quality) — measures ONLY what it was built to measure
Sync offsetCross-correlate audio envelope vs. mouth-motion signal; the offset with peak correlation is the measured lag
No-reference quality scoreA small model predicting human-perceived quality with no "correct" reference to diff against
Golden-set regression harnessFixed, deliberately varied inputs (angles, tones, voice types) run through every model/config change, diffed per-input against baseline
Borderline routingValues just inside a pass threshold route to human review too — "barely passed" is as untrustworthy as "barely failed"
Stratified human sampling100% of borderline/new-version renders + a small random baseline slice of clean ones, scored against a fixed rubric
The automation ceiling"Does this look natural" is semantic judgment, not a geometric measurement — no proxy metric closes that gap
Minimum viable capstone sliceOne model per stage, but the COMPLETE director → validate → execute → cache loop — narrow models, full architecture
Build orderSequence by what unblocks the most downstream work (caching/queue first, plan schema next) — not by what's easiest or most exciting
The five recurring invariantsJurisdiction, content-addressed caching, propose-never-execute, reject-vs-repair, scoped consent — reasserted at every layer on purpose
