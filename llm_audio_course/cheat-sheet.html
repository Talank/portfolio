Cheat Sheet

Every engine, knob, and pipeline recipe from the course in one place — how to synthesize non-robotic speech locally, free, and privately. Not a substitute for the lessons; a fast recall aid once you've done the labs. Organized in the same order as the dashboard.

Part 0 — Why Local Voice

ThingWhat it is
The data-leakage holeCloud TTS (edge-tts & friends) synthesizes on a remote server — every line of your text is sent to a third party. Inherent to the architecture, not a setting
The local guaranteeRun the model on your machine, so text never travels. "Nothing about your text leaves the box" — the whole point of the course
StrawHat NarratorThe capstone app: a speaker-labeled script → one distinct, emotion-tuned voice per line, rendered fully offline and played in the browser
Two-stage neural TTSAcoustic model (text → mel spectrogram) → vocoder (mel → waveform). How text becomes a voice
Phoneme / G2P front-endGrapheme-to-phoneme: the model pronounces phonemes, not letters — why it reads "SHISHISHI" as spelled-out letters
Timbre vs. prosodyTimbre = the voice's tone quality (modern models nail it); prosody = melody/rhythm (default is flat — the robotic part)

Part 2 — Killing the Robot

ThingWhat it is
The three prosody knobsRate (tempo, % offset) · Pitch (frequency, Hz offset) · Volume (amplitude, % offset). Same words, different feeling
Baseline + deltaPer-voice baseline (character identity, e.g. ("+13%","+12Hz","+0%")) + per-line delta (this line's feeling). Composed, clamped
Small-and-purposeful ruleA few % / Hz add life; ±40 makes a chipmunk or a drunk. Clamp so no line goes cartoonish
Emotion per LINEEmotion tracks what the line is ABOUT, not who speaks. Nami fears the problem, loves the solution — opposite deltas, same voice
Keyword classifierCount fear-words vs excited-words; strict majority wins, tie/none → neutral. Deterministic, inspectable, no model. Fear = quieter/tense; excited = louder/open
Casting = a tablespeaker → (voice/embedding, baseline). Vary the MODEL first (timbre) for distinctness; unknown speaker → safe default. Keep it STABLE
Laugh rewrite"SHISHISHI" → "Shi hi hi hi ha ha ha!" — surgical regex on the SPOKEN text only; keep the signature onset, guard real words ("shirt")
Non-verbal routingRecurring laugh → rewrite (cheap, deterministic). Rare hero beat → Bark, generate offline & FREEZE the best take
Text normalizationOrdered rules: "O(n^2)" → "big oh of n squared", numbers/acronyms/units said right. Specific rules BEFORE general. SPOKEN string only
Acronyms = a dictionaryAPI→spell, NASA→word: no rule predicts it. Curate data. Numbers need intent (200 = two hundred? two-oh-oh?)

Part 3 — Pipeline Engineering

ThingWhat it is
The sentence is the atomChunk on sentences: long enough for coherent prosody, short enough to cache/parallelize/control. Paragraphs drift; words are choppy
Chunk after normalizeNormalization dissolves "e.g."/decimals first, so the splitter faces clean text — only guards residual initials/titles ("Dr.")
Content-addressed cachingFilename = hash of everything affecting the sound (spoken text, voice, prosody, engine/version). The name IS the fingerprint
Key = sound-inputs onlyMiss a field (prosody) → silent STALE audio. Add a non-field (timestamp) → breaks reuse. Serialize with sorted keys for a stable hash
Skip-if-existsRender only cache-misses. Change one line → its key changes → only it re-renders. 10-min build → 5-sec incremental
The manifestJSON index: line → clip + metadata (speaker, display text, prosody, emotion, gap). Render↔playback contract; detects "0 missing"
Engine-version fieldIn every key = a deliberate GLOBAL invalidation switch. Bump it → full re-render. Leave it out → silent old/new mix
Batch renderEmbarrassingly parallel on top of the cache. BOUNDED worker pool (≈cores CPU / VRAM-fit GPU); unbounded → OOM/thrash
Retries: transient vs deterministicRetry transient (hiccups/timeouts) with backoff; flag deterministic (bad input) loudly. One bad line never aborts the batch
Idempotent + resumableInherited from the cache: a crash re-runs only unfinished work, never from zero
Browser playbackOne native <audio>, advance on the "ended" event. No library, no build — splicing became sequencing
Captions · gaps · duckingCaption from the manifest (synced free) · controllable pause between clips (silence = 4th knob) · SECOND element for music, volume DOWN on voice, UP in the gap

Part 4 — Privacy, Cloning & Ethics

ThingWhat it is
The airgap testPull the cable, run the full pipeline. If it works, nothing left the box — EMPIRICAL proof, stronger than any policy or code review
Eliminate trust vs place itCloud = trust a policy you can't verify (the server decrypts your text). Local = no third party to trust. The airgap proves the surface is zero
Pin offlinePre-download weights, force offline mode (HF_HUB_OFFLINE=1). No runtime fetch → passes airgap on a FRESH machine + reproducible
The sneaky callsModel auto-download on first use (fires once — a warm machine hides it), telemetry, update checks, CDN assets. Monitor outbound to catch them
Zero-shot cloningSpeaker encoder: short sample (6–10s) → embedding ("voiceprint") → conditions the synthesizer. Says new words the person never said
The sample is everythingClean, adequate-but-short, representative, matched conditions. Garbage in, garbage out — noise/reverb bake into every line
Local ≠ consentLocal protects the biometric DATA; it says nothing about whether the person AGREED. Two different problems
Consent: informed · scoped · beforeThey understand the act · bounded to specific uses (not a blank check) · a precondition, not after-the-fact forgiveness
ProvenanceDurable record per voice: owner, sample source, permitted uses, date. GATES usage (required field), enables REVOCATION & accountability
Disclosure + designing against misuseLabel synthetic audio honestly. Going local removes the external gatekeeper → build guardrails in & refuse the harmful uses yourself

Part 5 — Capstone: StrawHat Narrator

ThingWhat it is
Input shape[{speaker, text}, ...] — the exact episode structure. Reproduce the character-voiced narration locally & privately
The full per-line pathnormalize → rewrite laughs → chunk → cast → emotion (baseline+delta) → route → cache key → render → manifest → playback
Order is load-bearingNormalize before chunk before cache; cast (baseline) before compose (delta); render before play. Each stage consumes the previous output
Privacy wraps everythingenforce_airgap() around the whole render; cloned cast voices carry a required consent record. Built in, not bolted on
The compounding decisionSentence-as-atom made caching granular, emotion per-line, rendering parallel, AND playback trivial sequencing. One choice, everywhere
Extensions = a stage or a seamNew engine → a plugin behind synth(). New emotion → words + a delta. Cloning → a cast entry. Streaming → a queue. Never a rewrite
The one habitSmall pieces · clean seams · deliberate order. Sophistication becomes composition — ten small functions in a row, not one big thing
