Big-O Refresher + Pythonic Idioms

Every pattern after this page assumes you're fluent in the tools below. Skim fast if you already know them — the point is recognition speed, not novelty.

Big-O, the fast version

ComplexityTypical source~Max n for <1s
O(1) / O(log n)hash lookup, binary searchanything
O(n)single pass, sliding window, hashing10⁸
O(n log n)sort-based, heap-based10⁶–10⁷
O(n²)nested loop, DP over pairs~10⁴
O(2ⁿ) / O(n!)brute-force subsets/permutations, unpruned backtracking~20–25
In an interview, say the complexity and justify it ("left and right each move forward at most n times, so O(n) amortized despite the nested-looking loop") — the justification is what's actually being graded.

collections — the single highest-leverage import

from collections import Counter, defaultdict, deque

Counter(words)              # frequency map in one call; .most_common(k) for top-k
defaultdict(list)           # groups without checking "if key not in dict"
deque()                     # O(1) appendleft/popleft — a list is O(n) for these
Using a plain list as a queue (list.pop(0)) is O(n) per pop. Reach for deque the moment you need FIFO behavior — this alone silently turns an "O(n) BFS" solution into O(n²) if missed.

heapq — priority queues without a library

import heapq

heap = []
heapq.heappush(heap, val)
smallest = heapq.heappop(heap)          # heapq is always a MIN-heap
heapq.heappush(heap, -val)              # negate to simulate a max-heap
heapq.heapify(existing_list)            # O(n) in-place, cheaper than n pushes
heapq.nlargest(k, iterable)             # top-k in O(n log k), no manual heap needed

bisect — binary search you don't have to write

import bisect

bisect.bisect_left(sorted_list, x)      # first index where x could be inserted, keeping sort
bisect.insort(sorted_list, x)           # insert into a sorted list, O(n) but no re-sort

itertools — combinatorics in one line

from itertools import combinations, permutations, product, accumulate

combinations(items, r)      # choose r, order doesn't matter
permutations(items, r)      # choose r, order matters
product(a, b)               # cartesian product, replaces nested for-loops
list(accumulate(nums))      # running prefix sums in one call

Comprehensions, unpacking, and other reflexes

# comprehension over a manual loop + append
squares = [x * x for x in nums if x % 2 == 0]

# enumerate instead of range(len(...))
for i, val in enumerate(nums):
    ...

# zip to walk two sequences together
for a, b in zip(list1, list2):
    ...

# tuple/multiple assignment — no temp variable for swaps
left, right = right, left

# walrus operator — assign inside an expression (great inside while/if conditions)
while (line := stream.readline()):
    ...

# sort with a key instead of a custom comparator
items.sort(key=lambda x: (-x.priority, x.name))

# dict.get with a default instead of manual "if key in dict" checks
count = freq.get(ch, 0) + 1
None of this changes an algorithm's Big-O — it changes how fast and bug-free you write it under interview pressure, and interviewers explicitly notice fluent, idiomatic code as a signal.
