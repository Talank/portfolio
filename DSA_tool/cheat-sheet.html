Pattern Recognition Cheat Sheet

Keep this open during practice (and mentally rebuild it before a real interview). Read the problem, match it to a row, start talking through that pattern's approach.

PatternRecognize when…ComplexityGo-to structure
Two PointersSorted array/string, need a pair/triplet meeting a conditionO(n) or O(n log n)indices from both ends, or same-direction slow/fast
Sliding WindowContiguous subarray/substring, longest/shortest/count satisfying a conditionO(n)left/right pointers + hash map or running sum
Fast & Slow PointersCycle detection, find the middle, linked-list "kth from end"O(n)two pointers moving at different speeds
Merge IntervalsOverlapping ranges, scheduling/meeting conflictsO(n log n)sort by start, sweep and merge
Binary SearchSorted array, or a monotonic search space ("minimize the maximum", "find the boundary")O(log n)lo/hi/mid — watch off-by-one on the boundary condition
Hashing PatternsNeed O(1) lookup, "have I seen this", complements, groupingO(n)dict / set / Counter
Linked List ReversalReverse all or part of a list in placeO(n) time, O(1) spaceprev / curr / next pointer shuffle
Monotonic StackNext greater/smaller element, histogram/skyline problemsO(n)stack kept strictly increasing or decreasing
Queue / DequeSliding window max/min, level-order processingO(n)deque holding indices (monotonic) or values
Tree DFS / BFSTraverse/transform a tree, root-to-leaf pathsO(n)recursion (DFS) or explicit queue (BFS)
Binary Search TreesValidate/insert/search leveraging sorted orderO(log n) avg, O(n) worstinorder traversal of a BST is sorted — exploit that
Heaps / Top-Kk largest/smallest, merge k sorted lists, running medianO(n log k)min-heap or max-heap capped at size k
BacktrackingGenerate all subsets/permutations/combinations under constraintsO(2ⁿ) / O(n!), prunedrecursion + make a choice, recurse, undo the choice
GraphsConnectivity, shortest path (unweighted), dependency ordering, cycle detectionO(V + E)adjacency list + visited set; topo sort / union-find for ordering/grouping
Dynamic ProgrammingOptimal substructure + overlapping subproblems — "count the ways", "min/max cost to…"usually O(n) – O(n²)memoized recursion or bottom-up table; define the state first
GreedyA locally-optimal choice provably leads to the global optimum — often paired with sortingO(n log n)sort, then single pass making the locally best choice
Bit Manipulation"Single number" tricks, subset enumeration via bitmask, O(1) extra space requiredO(n) or O(1)XOR / AND / OR / shifts
TriePrefix search, autocomplete, "word dictionary" style lookupsO(L) per operation (L = word length)nested dict or fixed-size children array per node
If two rows both seem to fit, that's often the point of a company variant — many "hard" interview questions are two patterns stacked (e.g. sliding window + hashing, or DP + binary search on the answer).
