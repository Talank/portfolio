Cheat Sheet

Every command and manifest pattern from the course, in one scrollable page. Not a substitute for the lessons — a fast recall aid once you've actually done the labs. Grows as each Part gets written; organized in the same order as the dashboard.

Part 0 — From Process to Pipeline

CommandWhat it does
docker --versionConfirm the Docker CLI is installed (does NOT confirm the daemon is reachable)
docker infoThe real daemon-reachability check — errors clearly if dockerd is not running
docker run hello-worldPull and run the canonical "it works" test image
docker run -it <image> shRun a container and drop into an interactive shell inside it
docker ps / docker ps -aList running containers / all containers including stopped ones
sudo systemctl start dockerStart the Docker daemon on Linux (enable --now to also start on boot)

Part 1 — Docker Fundamentals

Command / InstructionWhat it does
docker imagesList local images (read-only templates) — never changes as containers run
FROM <base>Start a Dockerfile from an existing image's layer stack
RUN <cmd>Executes at BUILD time; result becomes a permanent image layer
COPY <src> <dest>Copy files from the build context into the image at build time
WORKDIR <path>Set the working directory for every following instruction
ENV KEY=valueBake a runtime environment variable into the image (visible in every container)
EXPOSE <port>Documentation only — does NOT publish the port; use -p at docker run
CMD ["cmd", "arg"]Default command executed at CONTAINER START (not build time); JSON-array form preferred
docker build -t <name> .Build an image from the Dockerfile in the current directory
docker run -d --name X -p H:C -e K=V imgRun detached, named, port-published, with an env var
docker logs [-f] [--tail N] <c>Read a container's stdout/stderr; -f follows live
docker exec -it <c> shInteractive shell inside a running container (last resort, after logs)
docker stop / docker killGraceful (SIGTERM + grace period) vs. immediate (SIGKILL) stop
docker rm <c>Remove a stopped container (add -f to force-remove a running one)

Part 2 — Building Good Images

Pattern / CommandWhat it does
Copy manifest, RUN install, THEN copy rest of sourceMaximizes build-cache reuse — dependency layer stays cached on source-only changes
FROM node:20 AS build ... COPY --from=buildMulti-stage build — only copy the finished artifact into the final, slim stage
ARGBuild-time-only variable; NOT automatically present in the running container
RUN --mount=type=secret,id=XUse a credential during build without ever writing it into a layer
docker history <image>See every layer's size and originating instruction
docker tag src:tag registry/ns/repo:tagAdd a new name for the same image content (no duplication)
docker push / docker pullUpload to / download from a registry
Tag with a commit SHA, never :latestMakes "what is running" and rollback unambiguous
@sha256:... (digest)Content-hash reference — cannot be reassigned to different content

Part 3 — Multi-Container Apps

Command / KeyWhat it does
docker network create <name>User-defined network — enables DNS-based reachability by container name
host.docker.internalReach the HOST machine (not another container) from inside a container
docker volume create <name>Named volume — Docker-managed, survives container removal
-v name:/path vs -v /host/path:/pathNamed volume (portable) vs. bind mount (tied to one host path)
docker compose up [-d]Create the stack's network, start every declared service
docker compose stopStop containers, keep them + network + volumes
docker compose down [-v]Stop + remove containers/network (add -v to also delete volumes — data loss)
depends_onControls START ORDER only — not a readiness guarantee

Part 4 — Kubernetes Fundamentals

Concept / CommandWhat it does
Declarative desired stateDeclare WHAT should be true; a controller continuously reconciles reality toward it
API serverThe single component every kubectl command and controller talks to
kubectl apply -f pod.yamlSubmit a manifest's desired state to the API server
PodSmallest deployable unit — one or more containers sharing network/storage, ephemeral by design
kind: Deployment, replicas: NContinuously maintains N healthy Pod replicas, replacing failed ones automatically
selector.matchLabelsHow a Deployment identifies its own Pods — by label, never by name
kind: ServiceStable name/address, load-balancing across whichever matching Pods are ready
kubectl get pods -l app=XList Pods matching a label

Part 5 — Kubernetes in Practice

Concept / CommandWhat it does
kind: ConfigMapNon-sensitive key-value config, referenced by name from a Pod template
kind: SecretSame shape as ConfigMap; base64-ENCODED (not encrypted) by default — better access control, not cryptographic protection alone
envFrom: configMapRef / secretKeyRefInject config/secret values as environment variables
kind: PersistentVolumeClaimRequest for durable storage that follows a Pod across node reschedules
storageClassNameReferences a StorageClass for automatic (dynamic) PV provisioning
kubectl describe pod <name>Full status + Events section — check FIRST for scheduling/pull failures
kubectl logs [--previous] [-c container]Container output; --previous for a crashed instance's last logs
kubectl exec -it <pod> -- shInteractive shell inside a Pod (last resort)
Pending / ImagePullBackOff / CrashLoopBackOffNot scheduled / image pull failed / container starts then keeps crashing

Part 6 — CI/CD Pipelines

Concept / SyntaxWhat it does
CIAutomatically build+test every push — value is speed of feedback
Continuous Delivery vs. DeploymentHuman approval gate before prod, vs. fully automatic deploy on pipeline pass
checkout → build → test → push → deployStandard stage order — each stage is a real gate for the next
on: push: branches: [main]Trigger on push to a specific branch
on: pull_requestTrigger on PR open/update — build+test only, before merge
uses: actions/checkout@v4Pre-built action — clones the triggering commit onto the runner
run: <shell cmd>Execute a raw command on the runner
${{ secrets.NAME }}Reference an encrypted secret — never written into the committed YAML
${{ github.sha }}The triggering commit SHA — used as the image tag, automating "never :latest"

Part 7 — Shipping It & Capstone

Concept / CommandWhat it does
strategy.rollingUpdate.maxUnavailableMax declared replicas allowed unavailable during a rollout
strategy.rollingUpdate.maxSurgeMax extra Pods allowed temporarily during a rollout
livenessProbeFailing this RESTARTS the container — "is it broken"
readinessProbeFailing this REMOVES it from Service traffic, no restart — "is it ready right now"
kubectl rollout status deployment/X --timeout=NsBlocks until the rollout genuinely completes — the real success signal, not just a returned command
kubectl rollout undo deployment/XFast rollback — reverts to the previous ReplicaSet, kept scaled to zero
Four-layer mapDocker (package) → Compose (one machine) → Kubernetes (many machines, reliable) → CI/CD (automate the whole path)
